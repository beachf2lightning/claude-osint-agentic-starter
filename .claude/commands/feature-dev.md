# Feature development workflow

You are implementing a feature in this repository. Follow `CLAUDE.md`.

## Required steps

1. Restate the feature and acceptance criteria.
2. Identify files likely to change.
3. Create a concise implementation plan.
4. Add or update tests for the desired behavior.
5. Implement a small vertical slice.
6. Run targeted tests.
7. Complete edge cases and error handling.
8. Update docs, examples, or CLI help if relevant.
9. Run the repository validation commands from `CLAUDE.md`.
10. Summarize:
    - What changed.
    - How to use it.
    - Tests and validation results.
    - Risks or follow-up work.

## Constraints

- Preserve existing behavior unless acceptance criteria require a change.
- Ask before adding new dependencies.
- Keep the first implementation simple and reviewable.
