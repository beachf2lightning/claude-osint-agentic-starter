# Test plan workflow

Create or improve a test plan for the requested change.

## Required output

1. Behavior under test.
2. Existing coverage found.
3. Missing coverage.
4. Recommended tests:
   - Unit tests.
   - Integration tests, if useful.
   - Edge cases.
   - Negative/error cases.
5. Minimal commands to run.

Do not implement the plan unless the user asks.
